# PhoneTypist - 系统设计文档

> iPhone 作为 Windows 语音输入设备

## 项目概述

**目标**：让 iPhone 手机作为 Windows 电脑的语音输入器，通过语音识别将文字发送到 Windows，并在光标位置自动输入。

**核心价值**：
- Windows 原生语音输入体验较差
- 配合 AI Agent 使用，语音比打字效率更高

## 技术栈

| 平台 | 技术选择 | 说明 |
|------|----------|------|
| iPhone | Swift (原生 App) | iOS 原生开发 |
| Windows | C# / WPF | Windows 托盘应用 |
| 通信协议 | WebSocket | 局域网实时通信 |
| QR码 | AVFoundation (扫描) + QRCoder (生成) | 配对机制 |

## 使用场景

**连接方式**：
- 同一局域网（同一 WiFi）
- Windows 打开个人热点共享给 iPhone

**配对流程**：
1. Windows 客户端启动，显示 QR码（包含 IP + 端口）
2. iPhone App 扫码，获取连接信息
3. WebSocket 连接建立
4. 配对信息保存，下次自动重连

### 功能模块

| 模块 | 技术 | 说明 |
|------|------|------|
| 语音识别 | SFSpeechRecognizer + AVAudioEngine | 按住麦克风按钮开始说话，实时显示转录文字 |
| 发送逻辑 | WebSocket (Starscream) | 点击发送按钮，将文字通过已建立的连接发送 |
| QR扫描 | AVFoundation (原生) | 首次配对时扫描 Windows 显示的 QR码 |
| 配对存储 | UserDefaults | 保存已配对的电脑信息（IP、端口、名称） |
| 连接状态 | 状态机 | 已连接/连接中/已断开 |

### 用户操作流程

1. 打开 App → 自动尝试连接已配对的电脑
2. 按住麦克风按钮 → 开始说话 → 实时看到文字
3. 松开按钮 → 文字保留在屏幕，可编辑修改
4. 点击发送 → 文字发送到 Windows，界面显示发送成功

### 功能模块

| 模块 | 技术 | 说明 |
|------|------|------|
| 托盘图标 | Hardcodet.NotifyIcon.Wpf | 显示状态图标，右键菜单，双击弹出窗口 |
| QR码生成 | QRCoder | 显示包含 IP+端口 的 QR码供 iPhone 扫码 |
| WebSocket服务器 | System.Net.WebSockets | 监听端口，接受 iPhone 连接，接收文本消息 |
| 键盘模拟 | InputSimulator | 收到文本后，模拟键盘输入到当前光标位置 |
| 状态管理 | 状态机 | 等待配对/已连接/已断开 |

### 工作流程

1. **启动** → 托盘图标出现，后台启动 WebSocket 服务器
2. **等待配对** → 双击图标显示 QR码窗口
3. **iPhone扫码配对** → WebSocket 连接建立，状态变为"已连接"
4. **接收文本** → 收到消息 → 用 InputSimulator 输入文字到当前应用
5. **显示最近接收** → 托盘菜单显示最近接收的文字

### 图标状态

- 🔵 蓝色：等待配对
- 🟢 绿色：已连接
- 🔴 红色：连接断开/错误

## 通信协议

### WebSocket 消息格式

使用 JSON 格式：

```json
{
  "type": "text",
  "content": "用户输入的文字内容",
  "timestamp": 1710000000000
}
```

### 配对 QR码内容

编码格式：`phonetypist://<ip>:<port>`

示例：`phonetypist://192.168.1.100:8765`

## MVP 功能边界

**包含**：
- iPhone 语音识别 → 文字发送 → Windows 键盘输入
- QR码配对机制
- 连接状态显示
- 托盘应用最小 UI

**不包含（后续可扩展）**：
- 剪贴板同步
- 反向发送（Windows → iPhone）
- 多台电脑支持
- 历史记录管理
